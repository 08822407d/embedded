# FACTS — 已验证事实台账(勿无故怀疑)

> 给 **AI agent 在设计 / 分析 / 实验时当事实基础**用的(不为人工阅读优化,按"好检索、可信、不被误读"组织)。
> 只放**已核实**的事实(实测 / 用户确认 / 数据手册 / 日志直接观测),每条都带**来源 + 日期**。
> **为什么有这张表**:调试反复失败时,模型容易开始怀疑基本事实(系统状态、传感器读数、接线…),把本来正确的地面真值也推翻,白费力、越走越偏。这里钉死的事实**默认为真**。
>
> ## 使用规则
> 1. **进表门槛**:必须"已验证"且写明来源 + 日期。假设 / 猜测 / 目标值**不进**这里(放 `REQUIREMENTS`「待确认」或 `STATUS`「开放问题」)。
> 2. **调试时**:把下表当地面真值。失败先怀疑 **代码 / 逻辑 / 参数 / 接线**,不要先怀疑这些已验证事实。
> 3. **推翻需要具体反证**:若观测与某条矛盾,**停下 → 点名是哪条(F‑x)→ 与用户或实测复核**;确认后**更新该条(换新来源/日期)**,既不要默默当它错、也不要默默盲信。
> 4. **来源失效**(改接线 / 换硬件 / 重标定 / 改固件)→ 立刻把相关条目标记为「待重验」或移除。
> 5. **易变值不钉死**:会随时间 / 官方更新而变的量(如工具链版本号),不当作事实记录,或显式注明其易变性,以免日后把过期值当真。
> 6. 本表对"某事是否已验证"具**权威性**;其它文档若与此冲突,以此为准并去同步它们。
