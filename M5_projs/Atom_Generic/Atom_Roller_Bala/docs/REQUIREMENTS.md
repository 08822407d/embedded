# REQUIREMENTS — 需求与验收

## 1. 一句话目标
做一个由 **M5Atom 控制、RollerCAN 作反作用轮**的**平面反作用轮倒立摆**:机体绕底部楔形棱在竖直平面内单自由度摆动,靠飞轮反作用力矩把机体稳定在"中轴竖直"的不稳定平衡点上。

## 项目性质与可行性
- **性质**:自动控制**入门性质**的工程项目 —— 以严谨为先,尊重数学 / CS 事实(详见根 `CLAUDE.md`「工作原则」)。
- **可行性**:同类反作用轮倒立摆自平衡已被他人成功实现,**目标确定可达**(见 `FACTS.md` F‑9)。失败按实现 / 参数 / 接线 / 标定问题排查,不归因于"原理不可能"。
- **范围**:硬件已组装、形状固定(以 `docs/assets/balance-geometry-sketch.png` 为准);本项目只做**固件开发 + 测试 + 测参数**,机械 / 电气搭建不在范围。
- **优先级**:在严谨前提下走**最快实现路径**(不镀金、不为教学而额外写理论);设计该记的照记(成本低)。

## 2. 系统形态
- 侧视几何见 `docs/assets/balance-geometry-sketch.png`。
- 机体自下而上:楔形底(棱朝下,是与桌面的唯一支点)→ 外壳方框 → 内部 RollerCAN 飞轮(圆)→ 顶部 M5Atom(主控)。
- 形式:反作用轮倒立摆(RWIP / 平面 Cubli 式)。飞轮**不触地**,纯靠角动量反作用矩平衡;对地接触仅底部楔形棱。

## 3. 硬件平台
- 主控:**M5Atom v1.0**(M5Stack,ESP32 / Arduino,板 `m5stack-atom`)。
- 姿态传感:**Atom 板载 MPU6886 IMU**(经 `M5Atom` 库 `M5.IMU` 读)。✅ 已确认
- 执行器:**M5Stack RollerCAN**,作反作用轮,经 `M5Unit-Roller` 库的 I2C 类 `UnitRollerI2C` 控制。
- 通信:**M5Atom ↔ RollerCAN 走 4 线 I2C**(Atom Grove 口 → RollerCAN 的 I2C 口)。✅ 已确认
- 上位机连接:开发机 ↔ 开发板 **单条 USB Type-C 有线**(烧录 + 串口读数据);机体本身**无无线 / 远控需求**。✅ 已确认
- 灯效:FastLED(机载 RGB)。
- **软件铁律**:所有控制与通信**优先使用官方库 API**(`M5Atom`、`M5Unit-Roller`),不裸写底层、不自造轮子。

## 4. 机械几何(已锁定)
- 楔形底**顶角 = 120°**(对中轴半顶角 α = 60°)。
- **平衡态**:中轴竖直 ⟂ 桌面,两斜面各与桌面成 **30°**。
- **静置态**(倒到一个斜面贴桌):中轴与桌面成 **60°**(= 距竖直偏 **30°**)。
- 推论:**距竖直 ±30° 为恢复区边界**;超过即贴到一个斜面静置(陷阱态)。

## 5. 功能需求(初稿)
- FR-1 **自平衡(最小成功)**:在恢复区内把机体稳到**中轴竖直(θ=0)**,且**飞轮角速度 → 0、机体角速度 → 0**(静稳竖直平衡),并持续保持。
- FR-2(可选 / 后续,**不属最小成功**)**起跳(swing-up)**:从静置态自主起跳、回到恢复区。
- *(抗扰动不作要求:不设专门抗扰指标,入门 PID 方案通常也不专门处理;稳定器对小扰动有隐含容忍,但不列为目标。)*
- (遥测 / 日志经 USB 串口输出到开发机,供查看与调参;非"机体远控"。)

## 6. 非功能 / 硬约束(初稿)
- 实时性:平衡环更新率需足够高(具体待定,典型 ≥ 200 Hz 量级)。
- 安全:飞轮转速 / RollerCAN 电流必须限幅防失控(库自带 `setSpeedMaxCurrent`/`setStallProtection` 等可用);判定跌出恢复区后安全停机(策略待定义)。
- (其余待补)

- **系统化的建模 / 控制理论教学材料**:暂不作需求(用户当前不需要学习向讲解;后续要学再补)。
- (其余待补)

## 8. 验收标准(初稿)
- AC-1:在恢复区内松手后,机体收敛到中轴竖直、飞轮与机体角速度趋零,并持续保持 ≥ N 秒不跌(N 待定)。
- (抗扰动不作验收项。)
- (其余待补)
