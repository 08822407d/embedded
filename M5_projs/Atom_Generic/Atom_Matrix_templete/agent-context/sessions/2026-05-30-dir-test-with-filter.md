## 实测结论（tools/run.log）
1. **滤波器有效**：电机转动期间 accel 原始角峰峰 **62.9°**(−5.5~57.4°)，融合 pf 峰峰仅 **4.7°**(σ0.63，咬在30±2°)。
   伪象抑制~93%，**0 次危险/断电**（上次在首个滑行段就误断电）。
2. **方向判定：反向电流(−)朝平衡**。正(+)使 pitch 增大(离B外)，负(−)使 pitch 减小(朝平衡0°)；
   `restoreSign=−1`，确认脉冲 −0.66°(预期<0)✓。**保底恢复/起摆朝平衡 = 给负电流。**
3. **陀螺零偏每次上电会漂**：gy 上次+0.10 → 本次+0.40°/s。**验证"现标"决策，勿硬编码零偏。**
4. **新隐患：高转速振动混叠**。220mA/~600rpm 时 gy 出现 ±30~46°/s 大摆动，近零均值故互补滤波仍稳；
   但将来**高频平衡控制**需开 MPU6886 片上 DLPF / 提采样率(现 50Hz)抗混叠。
5. **挣脱力矩不足**：340~400mA 机体也只从 B 偏 ~1~2° 又被重力拉回。**稳态电流做不了起摆**，
   需 Cubli 式**动量泵**（飞轮加速储能→急刹把动量灌给机体）。

## 🔴 起跳阈值表征实测（2026-05-30，无人值守，tools/run_breakaway.log / run_vin.log）
自动从 B 朝平衡方向 −400→−1200mA 逐级加流、短脉冲、脉冲间静置自复位，全程 0 保底触发、机体每次回 B：
- **升到硬件上限 1200mA 仍不脱阱**，朝平衡前进恒为 ~1~2°，**与命令电流无关**。
- **根因＝电机供电不足（非控制策略）**。三条铁证：
  1. **实际电流被钳在 ~85mA**（命令 400~1200mA 全一样），且随转速上升而下降（back-EMF 特征）；
  2. **飞轮峰值转速恒 ~580~600rpm**，各档命令一致（动量上限固定 → 反作用冲量固定 → 机体前进固定）；
  3. **Vin ≈ 4.5V**（怠速 4.61V，满载 4.49~4.62V **不掉压**）→ 是低电压受限、不是供电电流塌陷。
- 含义：电机只有 ~4.5V（Grove 5V 经压降）可用，既冲不出大电流也转不快。**在此供电下不可能起跳**，
  动量泵也意义有限。**必须先解决供电**（RollerCAN 是否有独立电机电源口、是否接了更高电压）——已问用户，待答。
- 已新增遥测：`motorVoltage()`/CSV `vin` 列 + motorInit 打印 Vin，便于后续确认供电。
