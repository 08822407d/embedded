# agent-context — 项目持久化上下文存储

本目录是本仓库的**持久化上下文存储系统**，随 Git 一起版本化、跟随仓库迁移。
用于记录 AI Agent 协同开发过程中产生的、**不适合写进代码、但必须长期留存**的信息。

## 为什么需要它

AI Agent（如 Claude Code）自身的"记忆 / 任务上下文"通常绑定在**本地项目路径**上。
一旦仓库被移动、改名、克隆到别的机器，这些上下文就会丢失。

本目录把这类信息**显式落盘到仓库内**，因此：

- 仓库移动 / 克隆 / 换机器 → 上下文不丢。
- 任何人（或任何 Agent）拉下仓库即可还原"为什么这么做"的完整背景。
- 与代码一起走 Git 历史，可追溯、可回滚、可 review。

## 目录结构

| 子目录 | 用途 |
|--------|------|
| `requirements/` | 原始需求记录。用户提出的原始诉求、目标、约束，按时间/编号归档。 |
| `decisions/`    | 决策记录（ADR 风格）。关键技术选型、取舍、原因。 |
| `protocols/`    | 硬件 / 通信协议。串口命令格式、寄存器、引脚定义等。 |
| `sessions/`     | 会话交接笔记。每段开发会话的进展、未完成项、下一步。 |
| `glossary.md`   | 领域术语表。统一项目内的概念命名。 |
| `INDEX.md`      | 总索引。一行一条，指向上面各文件，便于快速检索。 |

## 使用约定

- 一条信息一处来源，不重复粘贴；用相对链接互相引用。
- 文件用 Markdown，命名 `NNN-kebab-title.md`（NNN 为三位递增编号）。
- 凡新增文件，在 `INDEX.md` 加一行指针。
- 日期用绝对日期（YYYY-MM-DD），不要写"今天/昨天"。
- 这是**人和 Agent 都要读**的资料，写清楚"是什么 + 为什么"，而非流水账。
