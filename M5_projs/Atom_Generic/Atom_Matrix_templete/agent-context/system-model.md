# 系统模型与标定

> 屏动平衡系统的简化物理模型，以及 IMU 姿态角的实测标定。
> 关联需求：[001 屏动平衡系统](requirements/001-screen-balance-system.md)。
> 最近更新：2026-05-29（第 1 步标定）。

## 1. 简化物理模型

把整机抽象为**等边三角形截面的三棱柱**，立在底棱上：

- **底棱** = 柱体长轴 = **唯一旋转自由度**；反作用轮绕此轴施加力矩。系统为**单轴反作用轮倒立摆**。
- **对称轴** = “重心 → 底棱”连线。
- **倾角 θ** = 对称轴与铅垂线的夹角（倒向一侧为正、另一侧为负）。

三个特征状态（沿唯一自由度）：

| 状态 | 描述 | 倾角 θ |
|------|------|--------|
| 静止态 A | 一个侧面贴桌（倒向 A 侧）| −θr |
| **平衡态** | 对称轴竖直、⊥桌面（不稳定平衡，倒立点）| 0 |
| 静止态 B | 另一侧面贴桌（倒向 B 侧）| +θr |

几何：等边三角形下顶角 60° ⇒ 侧面距桌面 60°、距对称轴 30° ⇒ 翻倒行程 θr ≈ 30°（对称轴距竖直）。

## 2. IMU 姿态角标定（2026-05-29）

固件：每 50ms 读 MPU6886 加速度，静态换算
`pitch = atan2(-accX, hypot(accY,accZ))`，`roll = atan2(accY, accZ)`（单位 °）。
三状态各取 30 个静止样本平均：

| 状态 | pitch | roll |
|------|-------|------|
| 静止态 A | **−31.03°** | −177.18° |
| 平衡态 | **+0.25°** | −177.96° |
| 静止态 B | **+29.81°** | −177.60° |

抖动均 <1°（手扶平衡态略大）。

## 3. 结论（模型 ↔ IMU 映射）

- **可动轴 = pitch。** 两静止态间 pitch 摆动 −31°→+30°（行程 ~61°）；
  **roll 全程恒为 ≈ −177.6°（±0.4°）**，是与运动正交的长轴方向，控制中忽略。
- **平衡点 pitch₀ ≈ 0°。** 两静止态中点 = −0.61°，实测平衡态 +0.25°，一致（手扶抖动内）。
  IMU 安装很正，几乎无 pitch 偏置 → 控制目标角直接取 0°。
- **θr ≈ ±30°**（A 侧 −31°、B 侧 +30°，左右对称），即对称轴距竖直 ~30°、距桌面 ~60°。

映射关系：
```
θ ≈ pitch − pitch₀,   pitch₀ ≈ 0°
  pitch < 0  → 倒向 A 侧       pitch > 0  → 倒向 B 侧
  平衡(倒立)点: pitch = 0°
  工作区间: pitch ∈ [−31°, +30°]   两静止态相距 ~61°
被控量 = pitch；roll 恒定无关，可不参与控制。
```

## 4. 电机辨识实测发现（2026-05-29）
- **加速度 pitch 在飞轮转动时不可信**：飞轮转/滑行时有振动尖刺 + 持续反作用偏置（低通也去不掉），
  实测把 pitch 顶到 37–42°造成误判；同期**陀螺 gy 仅 ±6°/s ⇒ 机体并未转动**。
  → 真转动判定/看门狗**必须用陀螺**；加速度仅在飞轮≈0且稳时作绝对基准。
- **机体"挣脱力矩"**：≤100mA 推不动机体（飞轮飙到 ~500rpm 而机体不动），
  判向/起摆需更大电流越过 B 凹坑的重力恢复力矩。
- 详见会话 [2026-05-29 电机方向辨识](sessions/2026-05-29-motor-dir-test.md)。

## 6. 控制动力学模型（速度控制方案）——★作用量是**扭矩**（2026-05-31 确立）

### 动力学
- 机体绕底棱：`I_b·θ̈ = m·g·l·sinθ − τ`
  - `m·g·l·sinθ`：重力矩。几何上 **θ=0 是不稳定顶（势能最高，倒立点）**、θ=±30° 是稳定阱。
    线性化 θ≈0：`I_b·θ̈ ≈ m·g·l·θ − τ`（正反馈、不稳定，需反馈镇定）；静止态阱附近重力矩把机体**拉回阱**（回复矩）。
  - `τ`：电机给飞轮的力矩，其**反作用 −τ 驱动机体**。**τ 是唯一执行权威（作用量）。**
- 飞轮：`τ = I_w·(θ̈+ω̇) ≈ I_w·ω̇` ⇒ **τ ∝ 飞轮角加速度 ω̇ = 飞轮转速的变化率**。

### 速度控制方案下，扭矩怎么来
- 命令飞轮目标转速 `ω_cmd`，速度环（力矩上限=maxCurrent）驱动飞轮跟踪。
- `τ = I_w·ω̇`，其中 `ω̇ ≈ dω_cmd/dt`（跟踪良好时），且 `|τ| ≤ maxCurrent` 对应力矩（`T=Kt·I`）。
- **施加一个确定扭矩的两种方式**：① 命令**速度斜坡**，斜率=ω̇=扭矩；
  ② `setSpeedMaxCurrent(目标力矩)` + 命令一个大速度阶跃，让速度环以该力矩上限**全力加速** ⇒ 扭矩≈该 maxCurrent 力矩。
  **②最直接：maxCurrent(mA) ≈ 扭矩**（力矩 ∝ 电流）。封装 `motorSetSpeedMaxCurrent()` 已就绪。

### 关键纠正（框架）—— 推翻"探测转向"
- **作用量 = 扭矩 τ**，不是飞轮"转速"、更不是"转速方向"。**"探测电机往哪边转"是错误说法。**
- **起跳 = 施加足够大、朝平衡方向的扭矩**，克服静止态重力回复矩、把机体挣脱阱朝平衡推。
- 探测应**寻找"能起跳的扭矩"`τ_break`**：逐步增大扭矩(maxCurrent)、施加朝平衡方向，找机体**挣脱阈值**。
  方向(使θ减小的飞轮速度变化方向)是该扭矩的**属性**，由物理+一次符号确定；探测中若机体**朝外棱挣脱**即说明符号反、取反。
- 之前"逐步增大目标转速 SU"探的其实是**冲量/角动量**(maxCurrent 固定时加速时间随 SU 变长)，**不是扭矩**；
  挣脱阱靠**瞬时扭矩 > 回复矩**，故**探扭矩(maxCurrent)才对**。`挣脱条件: τ > 静止态阱重力回复矩峰值`。
- 旁证：§4 早已实测"≤100mA 推不动机体"——那就是扭矩(电流)不够、未达 τ_break。
