# 缓落控制：速度模式 vs 电流(力矩)模式 —— 研究与架构

> 起因：起跳越平衡后的"缓落/落地"用电流模式做效果不稳（-K·ω 阻尼刹车实测翻车）。
> 研究速度模式能否让缓落更简单，并保留电流模式平衡主回路。结论 + 已落地的可换架构。

## 一句话结论
- **平衡（钳在不稳定顶点）→ 电流(力矩)模式**：高带宽、低延迟，状态反馈 `τ=-K·x` 直接映射电流，最贴物理。保留。
- **缓落 / 动量管理 / 去饱和 → 速度模式更简单**：把"速度环"闭进电机，命令一个由动量守恒算出的
  目标飞轮转速，内环限流软逼近、自带防过冲整形。值得做成可换策略实测对比。
- 已落地：起跳策略新增 `SWINGUP_SPEED`（速度模式起跳+速度环动量接住缓落），与电流模式 CUBLI/IMPULSE 并列。

## 为什么电流模式做缓落难（已实测失败）
- 缓落本质是**速度问题**：要让机体到达对面静止态 A/B 时角速度 θ̇_b → 0（不是位置问题）。
- 电流模式只有"轮力矩"一个输出，要把 θ̇_b 归零必须**自己闭一个速度环**；而力矩的积分=飞轮转速会**饱和**。
- `-K·ω` 阻尼刹车失败根因（run_cubli4 翻到 pitch=84°）：**反作用轮刹车不耗能**，机体动量被搬进飞轮，
  飞轮存的动量随后**回流**又把机体翻过去。既要调 torque 增益又要防饱和，难调且易翻。

## 为什么速度模式可能更简单
- 速度模式 = 把"速度源"交给电机内环。你命令飞轮转速，内环用**最大电流(限幅)平滑逼近** → 自带软逼近/防过冲的力矩整形。
- 缓落的物理 = 机体到达静止态瞬间，把机体角动量灌进飞轮使 θ̇_b=0。由动量守恒，所需飞轮转速增量（相对机体）：
  ```
  Δω_w ≈ (I_b + I_w)/I_w · θ̇_b
  目标转速 catchTgt = 当前飞轮转速 + spinSign · J_RATIO · |θ̇_b|
  ```
  把 catchTgt 命令给速度环，它自动产出**限流、平滑、收敛**的力矩曲线；每帧按残余 θ̇_b 重算 → 机体越慢增量越小，自然软收敛。
- 即"把环闭在电机里"：用"算一个动量目标转速 + 让伺服过去"替代"调一个跟饱和搏斗的力矩环"。**这是简化所在。**
- 落定后机体在 A/B 是**稳定**势阱（gravity 自持）；飞轮虽满载，可在静止态用**斜坡缓退转速去饱和**，
  退速的小反力矩由势阱吸收。正合 decision 002 早写的"速度模式用于外环动量管理/去饱和"。

## 代价 / 边界（诚实说明）
- 速度模式**不改物理**：飞轮仍会饱和、仍需事后去饱和步骤。它只是把环闭的位置挪进电机，让缓落实现更平滑简单。
- 速度环有**相位滞后** → 不适合倒立点平衡的快回路。⇒ 平衡仍用电流模式。
- `J_RATIO = (I_b+I_w)/I_w·折算` 未标定（缺机体/飞轮惯量）→ 代码给**初值 4.0**，须实测拟合（°/s→rpm）。
- **模式运行中不可切**（decision 002：切模式要先撒手且慢）。⇒ 缓落用速度模式时，让**整个起跳测试跑在速度模式**
  （蓄能/起跳/缓落/去饱和同模式）；电流模式平衡控制器属**另一个 run / 另一入口**，二者不在同一次运行里切模式。

## 已落地的架构（可换）
- `motor.*`：新增 `MotorMode{CUR,SPD}` + `motorMode()`；init 记录模式；**`motorStop()/motorPowerOff()` 改 mode-aware**
  （速度模式下 `setSpeed(0)`，修掉了速度模式误发 `setCurrent(0)` 的潜伏 bug）。
- `dir_test.*`：起跳策略枚举加 `SWINGUP_SPEED`；`swingUpTest()` 分派器**按策略自动把电机 init 成对应模式**
  （SPEED→`motorInitSpeed`，其余→`motorInit`）。共享保底 `poweredDangerStop` 经 mode-aware 的 stop/powerOff，两模式都安全。
- **切换方式 = 一行**：`swingUpSetStrategy(SWINGUP_SPEED)`（电流流程原样保留：默认 `SWINGUP_CUBLI`）。
### `SWINGUP_SPEED` = 全速度模式 起跳+缓落+兜底（用户定：运行中绝不切模式，前置探测到落地全程速度模式）
`swingUpSpeed()` 四段（前置 `prepAtRest` 仅读 IMU、模式无关；电机 init 即速度模式）：
- **[0] 速度模式驱动/供电自检**：命令 +200rpm 看飞轮是否响应（速度环工作=供电/驱动正常；替代电流模式供电探测）。
- **[A/B] 探测**：从低速**单调抬高目标转速**——速度环到目标后力矩=0，必须持续 ramp 才有持续反作用力矩推机体。
  记 ① `baseSpd`(机体开始朝平衡前进=脱阱基准/"合适速度") ② 用固定**"改变速度的间隔"`RAMP_STEP`(rpm/帧)**持续抬高
  ③ `minCross`(最小越平衡目标转速=起跳所需最小控制量)。全程 `speedDanger` 危险即 setSpeed0+断电。
- **[C] 缓落**：越平衡**立即**逐帧 `catchTgt = 飞轮转速 + spinSign·J_RATIO·|θ̇_b|`（动量守恒接住，机体越慢增量越小自然软收敛），
  落定对面静止态后**斜坡退飞轮转速去饱和**（稳定势阱吸收小反力矩）。
- **[D] 兜底(保留)**：收尾若 |面内 pitch|>40°(翻越) → 垂直角<10° 面内**速度击杀试救**、≥10° 三维翻滚断电不救。
- **参数 `RAMP_START/RAMP_STEP/RAMP_CAP/baseSpd/minCross/J_RATIO` 为探测/估算量，待实测调**（详见函数注释）。
