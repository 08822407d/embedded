# 渐进逼近极限的安全探测规范（2026-05-30 起摆翻倒事故后总结）

> 用途：任何"找极限/调能量"的电机探测（挣脱、起摆、起跳）都按本规范，**逐步逼近临界、留裕度、不越线**。
> 起因：一次起摆把机体翻出平面、停在不可恢复姿态。根因与对策见下。配合 [playbook 第5节排查表](reaction-wheel-pendulum-playbook.md)。

## 一、事故根因（确认）
1. **5V 时"靠欠功率而安全"**：电机最大电流也推不动机体 → 参数开多大都不翻；这种安全是**假象**，来自欠功率。
2. **换 12V（authority ≈3×）没缩参数倍率**：驱动用的是**绝对 mA**，同一命令现在做 3× 的功 → 一下过临界。
3. **中途换策略且没重调参**：挣脱/弹跳 → Cubli 蓄能急刹（更猛），又没从低重新 ramp → 直接上满能量。

## 二、关键定量规则（为什么反应式保底拦不住）
- **保底反应时间 T = 消抖帧数 × tick**（现 3×20ms = 60ms）。机体在 T 内移动 ≈ V·T。
- 反应式保底要有效，必须：**V_safe < 安全裕度 M / T**。
  M ≈ 4°（静止态 B(+30) 到翻倒棱(~+34/+42)）、T = 60ms → **V_safe ≈ 60~70°/s**。
- 事故时 V = **1261°/s**（25°/帧），是 V_safe 的 ~19 倍 → 60ms 内冲 75°，从 0 越到 +38° 翻掉。
- 推论：
  - **探测期机体角速度必须钳在 ~60°/s 量级**，反应式保底才管用。
  - **需要高速的执行机动（真起摆越平衡）不能靠反应式保底**，必须**预测式急刹**（按当前角度+角速度预测落点，提前在合适相位刹），本质需顶点附近**平衡控制器**接住。

## 三、渐进逼近极限的探测框架（重置恢复后照此实现）
1. **相对 authority 表参数**：先 `powerDetectTest` 测 authority（飞轮顶速）；驱动用 **authority 的百分比/相对量**表达，不用绝对 mA。供电一变，同一百分比自动缩放 → **换电压不会突然过猛**。
2. **永远从低地板起 ramp**：每次从 effort_floor（很小）开始，**小步**增大；绝不一上来给大值。
3. **任何变更后重新归零基线**：换供电 / 换策略·算法 / 改关键参数 → effort **回到地板重新 ramp**，**绝不沿用别工况的高参数**（事故第2、3条的直接对策）。
4. **响应梯度自适应步长**：每步测响应（峰值偏移 E、峰值角速度 V）；估梯度 dE/d(effort)；
   **让下一步预测 E_next ≤ 临界 × 裕度系数(如 0.7)**；越接近临界步长越小（防"最后一步跨过去"）。
5. **速度天花板**：任何一步 V 超 V_safe(~60°/s) → **立即停 ramp / 回退**（再加就要失控）。
6. **逼近即停、留裕度**：E 达到临界的 ~70% 就**停止增大**，报告"逼近到的极限值 = X"，**不越临界**。
7. **侧向严守**：全程侧向（出平面）看门狗严格——实测确认**侧向倾倒不可恢复**（满动量也救不回）。
8. **步间自复位校验**：每步前确认机体已回到已知安全静止态（A/B），否则中止。

## 四、伪代码（探测一个极限）
```
authority = powerDetectTest()              // 相对基准
effort = EFFORT_FLOOR                       // 从低地板
prevE = 0
loop:
  if not atKnownRest(): abort
  E, V, L = doOneStep(effort * authority)   // 单步机动并测响应
  if L > LAT_WARN: abort                     // 侧向不可恢复
  if V > V_SAFE: stop("速度逼近上限")        // 反应保底失效前停
  if E >= 0.7 * E_CRIT: stop("已逼近极限")   // 留裕度
  grad = (E - prevE) / step                  // 响应梯度
  step = min(step_max, (0.7*E_CRIT - E)/grad) // 让下一步预测不越线
  prevE = E; effort += step
report 安全极限 = effort
```

## 五、与执行机动（起摆）的分工
- **探测**：慢、找极限、不越线（本规范）。产出：挣脱阈值、最小越平衡能量、各档响应曲线。
- **执行**（真起摆需高速越平衡）：用探测得到的最小能量，且**预测式**控制落点（非反应式），顶点附近交给**平衡控制器**接住。**不要在执行阶段才第一次上大能量**。
