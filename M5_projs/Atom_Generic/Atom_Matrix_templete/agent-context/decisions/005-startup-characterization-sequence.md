# 005 — 启动自动表征序列（固定探测流程）

- 日期：2026-05-30
- 状态：设计（待机体复位后实现/实测）

## 背景
需要开机后跑一套**固定的自动表征流程**：先确认供电，再（若供电足以起跳）逐侧探测临界值。
所有探测必须套用 [渐进逼近极限规范](../references/safe-limit-probing.md)（从低 ramp、速度钳、逼近即停、相对 authority、侧向严守）。

## 流程（状态机）
```
boot →
[1] 供电探测 powerDetectTest()
    - 不足(仅5V，顶速≈600) → 报"供电不足，不起跳"，停。
    - 充足(12V，顶速远超阈值，足以起跳) → 继续。
[2] 识别当前静止态 S (A/B)
    - 非 A/B（翻倒/未知）→ 报"需人工复位到 A/B"，停。
[2.5] **击杀式方向探测**（安全确认电机往哪转才朝平衡）：
    - 在预期朝平衡方向(-st)给一个**小冲量**(~250mA/160ms，朝外棱仅偏~0.6°，安全)；
    - 若机体反而**朝外棱(危险边界)运动 → 转向相反**(应取 +st)；速度钳+逼近外棱即停。
    - 结果与预期不符/不明确 → **停止**（方向未确认前不探边界/不起跳）。`directionProbeKick()`。
[3] 对当前态 S 做临界值探测（两个旋转方向，渐进逼近，绝不越线）：
    - 方向①「朝平衡」(driveSign=-st)：ramp 找"接近越平衡(起跳)"的临界能量；逼近即停(不真越，留裕度)。
    - 方向②「朝外棱」(spinSign=+st，危险不可恢复侧)：ramp 找"接近翻倒"的临界值；逼近即停(绝不翻)。
    - 全程按 safe-limit-probing：相对 authority 表参、从低小步 ramp、速度钳 V_safe、步间自复位、侧向看门狗。
    - 记录 S 侧两个方向的临界值，标记 S 已探测。
[4] 换边：**仅当机体确实到了另一边(A↔B)才探对面**。
    - 安全探测"逼近即停"不会自己越过 → 默认一次只探一边。
    - 探对面前提：机体正停在对面（人工放过去，或将来用"受控越平衡+预测式落地"送过去）。
    - 机体到达对面之前，**禁止探对面**（此刻对面临界无从测）。
[5] A、B 两侧都探完 → 输出完整临界表（每侧×每方向），结束。
```

## 关键原则
- **必区分旋转方向**：每侧两方向物理意义相反——一朝平衡(可起跳)、一朝外棱(不可恢复危险)，临界值不同。
- **A/B 不对称**：套件物理偏心等使两侧"到平衡"和"到翻倒"的临界都不同，**分别探、不假设对称**（实测：A=-31°/B=+30°已不对称）。
- **就近探测**：机体当前在哪侧就先探哪侧（要探某侧，机体必须正停在该侧）。
- **安全第一**：探测期绝不翻倒；逼近即停留裕度。

## 实测（2026-05-31，run_startup）✅ 前半段全通过、渐进框架验证有效
- [1] I²C 自检 IMU/电机均 OK；[2] 供电探测 1860rpm→12V 充足；[3] 识别=A(-31.4°)。
- [4] A 侧危险边界探测（朝外棱 -电流 渐进 -40→-415mA）：**朝外偏移平稳增长 0.4°→2.67°、角速度全程 ≤33°/s(<60钳)，
  到 2.5° 偏移即停，留 ~1° 裕度，全程未翻** → **A 侧危险边界 ≈ -415mA**。
  对比 Cubli 满能量 1261°/s 一冲就翻：**"从低ramp+速度钳+逼近即停"框架实测可用**。
- 注：-415mA 是"朝外偏移达 2.5°"的保守安全限(距外棱~1°)，非真实翻倒电流(更高)；作"朝外勿超"运行上限用。
- 待办：[5] 起跳到对面+探 B 侧边界——需先解决"安全越平衡落地"；或人工把机体放到 B 再跑一次即可得 B 侧边界。

## 待定参数（实现前定）
- V_safe 速度钳：≈60°/s（=外棱裕度4° ÷ 保底反应60ms）。
- 逼近停止裕度：朝平衡停在距平衡 ~5° 处；朝外棱停在距外棱 ~2° 处（外棱仅 3-4°，须很保守）。
- effort 地板/步长：相对 authority 的百分比（如地板 5%、步 +5% 并随梯度收缩）。
- 朝外棱方向尤其谨慎：外棱距静止态仅 ~3-4°，速度钳+小步是关键。
