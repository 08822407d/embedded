# 006 — 固定流程契约：全速度模式 初始化→监视→运动探测→起跳落地

- 日期：2026-05-31
- 状态：**已锁定（基线 v1）**。后续讨论即便口头/听写模糊，也以本文件为准——
  **不得擅自拆掉这套固定流程重做**；要改先改本文件并记录。
- 关系：在速度模式下**取代** [005 启动表征序列](005-startup-characterization-sequence.md)（005 为电流模式版）。

## 全局约束（必守）
- **全程速度模式**，运行中**绝不切换**控制模式（见 [002](002-motor-control-strategy.md)：切模式要先撒手且慢）。
- **渐进逼近规范**（见 [safe-limit-probing](../references/safe-limit-probing.md)）：保守起步、逐轮上调、变更重归零、留裕度。
- **关键量入日志**：统一 CSV schema（同 `sampleAndLog`），全速率，供离线计算回填工程。

## 阶段一 · 初始化与自检（必做）
1. I²C 通信初始化（IMU=Wire1 / 电机=Wire）+ **双总线通信自检**（ACK）。失败即停。
2. 电机初始化为**速度模式**（`motorInitSpeed`）；Vin/过压自检，不达标即停。

## 阶段二 · 持续传感监视（必做，贯穿全程 + 作终点态）
3. **全速率**读 IMU 全部数据 —— 读哪些、CSV schema **按现有 `sampleAndLog`，不改**。
4. 按**时间窗（1s）聚合判定**系统姿态状态 —— **按现有 `attitudeReportTick`，不改**。
   窗口仅用于"状态判定"(抗单帧噪声误判)，**不降低逐帧打印频率**。

## 阶段三 · 运动探测（多轮，逐轮增大，找"方向 + 参数"）
5. **运行时自动识别**初始姿态 A/B（`prepAtRest`，无需重烧/写死）。
6. **多轮探测**找正确起跳方向与参数：
   - 每轮**一次性**施加**一个固定目标转速**（**不是轮内逐渐加速**），**轮间**逐步增大。
   - 每轮开始前**必须已停稳回到静止态 A/B** 才施加（防同相位泵能/震荡累积）。【B】
   - 观察机体是否越平衡；记录朝平衡前进量/峰值/落点（入日志）。
   - **未越平衡 → 回落消能**：机体回落时主动施加**与起跳同向**的反作用力矩，抵消回落动能，
     **防止回落过冲越过本侧静止态外棱而翻倒**（外棱距静止态仅 3-4°，余量稍大即翻——核心风险）。【C·未越】

## 阶段四 · 起跳与落地（探出正确参数后）
7. **起跳 = 一次性输入**该合适参数（**非渐进**），使机体越平衡。
   （实践中：阶段三里"第一轮成功越平衡"的那一档参数即为它；记录该参数以便后续直接复用，不必每次重探。）
8. **落地缓落**：机体**越过平衡到对面** → **反向消能/动量接住**（与起跳**反向**的转动惯量消余量），
   平稳落到对面静止态；随后**斜坡缓退飞轮转速去饱和**。【C·已越】

## 贯穿全程的硬保底
- **危险看门狗（硬断电）= 出平面/横向轴**（飞轮无 authority 的那个维度，代码里 `lat`）越界 → 断电、不救。【A】
- **面内（pitch）越外棱不算硬断电**：由阶段三/四的"消能"预防；万一仍面内翻越，交**面内击杀自救** [D]
  （`|pitch|>40°` 翻越 且 `|lat|<10°` 仍在平面 → 击杀试救；`|lat|≥10°` → 断电不救）。
- **飞轮超速**（runaway）→ 硬断电（保留）。

## 终点态
- 无论**成功 / 失败 / 中止**，结尾一律回到「**电机断电 + 阶段二持续监视**」。
