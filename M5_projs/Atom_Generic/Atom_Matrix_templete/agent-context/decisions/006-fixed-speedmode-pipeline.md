## 修订（2026-05-31 晚，用户校正）—— 现行目标已简化
- **危险处理简化**：**超出平衡 40°(|pitch|>40) 或 横向/超速 → 立即断电、仅持续监视、不再自救**
  （自救在侧向不约束下无效，见文末；rescue 代码保留但当前流程不调用）。
- **开机目标缩小为"方向表征"**：只判定+记录"使机体朝平衡起跳的飞轮**速度变化方向**"，记录后**终止**
  （断电+监视）。入口 `probeSwingUpDirection()`，结果存 `g_swingUpDir`(+1/-1)。
- **关键物理校正**：机体运动方向由**反作用力矩**决定；力矩由**飞轮角加速度=速度变化 dω/dt 的方向**决定，
  **不是飞轮当前转速方向**。⇒ 控制量是"**速度变化方向**"。`speedDanger` 也据此用 dω 思路（消能器 target=ω−K·θ̇ 本就如此）。
- 下方"阶段三/四 起跳+落地+自救"为**完整方案存档**，待机械单轴约束就绪后再启用；当前先把方向测准记下。

## 全局约束（必守）
- **全程速度模式**，运行中**绝不切换**控制模式（见 [002](002-motor-control-strategy.md)：切模式要先撒手且慢）。
- **渐进逼近规范**（见 [safe-limit-probing](../references/safe-limit-probing.md)）：保守起步、逐轮上调、变更重归零、留裕度。
- **关键量入日志**：统一 CSV schema（同 `sampleAndLog`），全速率，供离线计算回填工程。

## 阶段一 · 初始化与自检（必做）
1. I²C 通信初始化（IMU=Wire1 / 电机=Wire）+ **双总线通信自检**（ACK）。失败即停。
2. 电机初始化为**速度模式**（`motorInitSpeed`）；Vin/过压自检，不达标即停。

## 阶段二 · 持续传感监视（必做，贯穿全程 + 作终点态）
3. **全速率**读 IMU 全部数据 —— 读哪些、CSV schema **按现有 `sampleAndLog`，不改**。
4. 按**时间窗（1s）聚合判定**系统姿态状态 —— **按现有 `attitudeReportTick`，不改**。
   窗口仅用于"状态判定"(抗单帧噪声误判)，**不降低逐帧打印频率**。

## 阶段三 · 运动探测（多轮，逐轮增大，找"方向 + 参数"）
5. **运行时自动识别**初始姿态 A/B（`prepAtRest`，无需重烧/写死）。
6. **多轮探测**找正确起跳方向与参数：
   - 每轮**一次性**施加**一个固定目标转速**（**不是轮内逐渐加速**），**轮间**逐步增大。
   - 每轮开始前**必须已稳定回位 ≥0.5s** 才施加（`confirmSettledAtRest`）：pitch 入带 **且 横向(出平面)≈0
     且 角速度小，连续保持 0.5s**——单帧入带不算，防震荡未停/未察觉的侧向失稳就开下一轮。【B 加严】
   - **轮间增量= 模型自适应、稳步、不贪心**：speed 模式 torque 不可直接算，故用**经验拟合 输入T→输出"朝平衡峰值前进 adv"**
     隐式代表 torque。每轮读 `adv` 峰值，按 `adv-vs-T` 局部斜率把"想再前进 X°"换算成 ΔT；
     **每轮最多再追加 ADV_INC_CAP(=8°) 前进量**，ΔT 限幅——绝不一次跳到"最合适值"，避免某轮过猛冲过外棱翻倒不可复。【新增】
   - 观察机体是否越平衡；记录朝平衡前进量/峰值/落点（入日志，供离线拟合回填）。
   - 异常：某轮机体**朝外棱(远离平衡)动** → 疑方向反/失稳 → 终止探测转兜底（不再贪进）。【新增】
   - **未越平衡 → 回落消能**：机体回落时主动施加**与起跳同向**的反作用力矩，抵消回落动能，
     **防止回落过冲越过本侧静止态外棱而翻倒**（外棱距静止态仅 3-4°，余量稍大即翻——核心风险）。【C·未越】

## 阶段四 · 起跳与落地（探出正确参数后）
7. **起跳 = 一次性输入**该合适参数（**非渐进**），使机体越平衡。
   （实践中：阶段三里"第一轮成功越平衡"的那一档参数即为它；记录该参数以便后续直接复用，不必每次重探。）
8. **落地缓落**：机体**越过平衡到对面** → **反向消能/动量接住**（与起跳**反向**的转动惯量消余量），
   平稳落到对面静止态；随后**斜坡缓退飞轮转速去饱和**。【C·已越】

## 贯穿全程的硬保底
- **危险看门狗（硬断电）= 出平面/横向轴**（飞轮无 authority 的那个维度，代码里 `lat`）越界 → 断电、不救。【A】
- **面内（pitch）越外棱不算硬断电**：由阶段三/四的"消能"预防；万一仍面内翻越，交**面内击杀自救** [D]
  （`|pitch|>40°` 翻越 且 `|lat|<10°` 仍在平面 → 击杀试救；`|lat|≥10°` → 断电不救）。
- **飞轮超速**（runaway）→ 硬断电（保留）。

## 终点态
- 无论**成功 / 失败 / 中止**，结尾一律回到「**电机断电 + 阶段二持续监视**」。

## 实测注意（run_006a / run_006c）
- 机体易**横向(出平面)失稳**。反作用轮**对横向无 authority**，横向一歪整套就推进不了。
- **run_006c（从 B 实跑起跳，控制逻辑全部验证通过）**：
  - ✅ 自适应模型探测生效：轮1 200rpm→前进8.9°、模型算 ΔT=179→379rpm、轮2 379rpm→前进11.8°。
  - ✅ 横向硬断电正确触发：轮2 中机体 lat 冲到 42° → 立即断电；机体随后倒到第三面(pitch≈83°/lat≈−2°)。
  - ❌ **真正 blocker = 机械：机体侧向自由、一推就出平面倒**。代码无问题。
- **结论：必须给机体加单轴转轴/轴承，约束成只能在 pitch 平面内转**，否则起跳永远推不上去。
- 旁证：机体在多次烧录间于 B↔第三面(83°)间反复跳变（无电机驱动），疑 USB/供电硬线张力扰动 + 侧向不稳。

  **侧向不约束时，连"自救"也失效**。⇒ 机械单轴约束是先决条件，控制端已尽力。
