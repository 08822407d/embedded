## Headless-Only Tool Invocation

`stm32cubeidec.exe` can build without a GUI only when an Eclipse headless application is specified. A bare `stm32cubeidec.exe --help` was unsuitable: it fell back to the default IDE application and left a process with the title `workspace_1.19.0 - STM32CubeIDE`; that process was immediately stopped and was never used for the build.

The verified non-GUI probe was:

```powershell
& "C:\Program Files\STMicroelectronics\STM32CubeIDE_1.19.0\STM32CubeIDE\stm32cubeidec.exe" `
  --launcher.suppressErrors -nosplash `
  -application org.eclipse.cdt.managedbuilder.core.headlessbuild `
  -data "$env:TEMP\codex-stm32cubeide-profiler-20260710-1019" -help
```

It printed the `headlessbuild` usage text and exited. The successful build below used the same `-nosplash -application ...headlessbuild` form. A process check after completion found no `stm32cubeide`, `stm32cubeidec`, `java`, or `javaw` process.

## Build Trace and Result

The clean temporary Eclipse workspace was:

```text
C:\Windows\TEMP\codex-stm32cubeide-profiler-build-20260710-1023
```

Initial import/build used the requested Debug configuration. It failed once during compilation because `mp_one_touch_tuning.h` includes `arm_math.h`, but the generated Debug include set omitted `GM16020-06_profile\Drivers\CMSIS\DSP\Include`. The header was present in the generated tree; neither HAL/CMSIS sources nor the Motor Profiler archive was missing.

Rather than hand-editing the ignored `.cproject`, the retry used the headless builder's transient `-I` option. This preserves the generated tree and makes the requirement explicit for every reproducible command:

```powershell
& "C:\Program Files\STMicroelectronics\STM32CubeIDE_1.19.0\STM32CubeIDE\stm32cubeidec.exe" `
  --launcher.suppressErrors -nosplash `
  -application org.eclipse.cdt.managedbuilder.core.headlessbuild `
  -data "$env:TEMP\codex-stm32cubeide-profiler-build-20260710-1023" `
  -no-indexer `
  -I "C:\Users\cheyh\Desktop\embedded\STM32\Nucleo-G431RB\IHM16M1_test\GM16020-06_profile\Drivers\CMSIS\DSP\Include" `
  -cleanBuild "GM16020-06_profile/Debug" -verbose -printErrorMarkers
```

Result: `Headless build completed successfully`, 0 errors, 0 warnings. The final link command used `-l:libmp-G4-IAR_ARMv7-M.a` and `STM32G431RBTX_FLASH.ld` as expected.
