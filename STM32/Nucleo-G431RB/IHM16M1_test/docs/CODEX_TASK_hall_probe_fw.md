## 目标
1. **主目标**:开发并烧录一套**只读传感器**固件,读 IHM16M1 上电机的 3 路霍尔,**手转转子时计数**,定出**极对数**(当前旁证:齿槽 ≈22≈24、6 槽 → 大概率 **4 对极**,需实测确认)。
2. **次目标(前瞻,能做则做)**:把"读霍尔 → 霍尔状态序列 / 电角度估计"链路打通并可实时观测(为后续用轴上指针看角度控制打基础)。
3. **全程不驱动电机**:见红线。

## ⛔ 红线(硬件安全,最高优先,违反即停并升级给 CC)
你(codex)可能不了解本项目硬件背景,以下红线**必须无条件遵守**,任何一条不确定就停下走 §-1 问,别赌:

1. **这是"只读传感器"固件,严禁任何驱动电机的行为**:
   - **绝不**配置/使能电机 PWM 定时器的**输出**(TIMx CH/CHN 到栅极);**绝不**使能 STSPIN830 功率级(EN/ENable、任何 gate-driver 使能脚);**绝不**驱动相桥臂/使能脚。
   - 凡控制功率级的引脚,固件要么**保持复位默认(输入/Hi-Z)**、要么**显式拉到"禁用"电平**(按 STSPIN830 数据手册的安全态),确保上电即安全、不会误使能。
2. **测试期间电机母线电源必须断开**(拔掉 DC 桶插座 / 母线端子),**只用 USB 给 Nucleo 供电**。烧录前经 HITL 让用户确认母线已断。
3. **若查明"不给电机母线上电就无法为霍尔供电"** → **立即停,升级给 CC(见 §-1 升级),不得自行给母线上电**。给母线上电改变安全边界,须由 CC + 用户在了解全貌后决定,不是你能定的。
4. **烧录授权**:烧录属项目红线,**仅允许烧录本诊断固件**,且**每次烧录前经 HITL 让用户确认"已授权 + 母线已断 + 目标板正确"**。授权会话内的多次迭代重烧可继续,但**每次都要维持上述安全不变量**。
5. **转子只能手动拨动**,严禁任何通电旋转。
6. **认准目标板再烧**:只烧 **NUCLEO-G431RB、ST-Link SN `002A00403234510E33353533`**(F-3);同机可能并存 Espressif 板,**别烧错**。用 SWD 连接时按 SN 指定。
7. **不碰 option bytes / RDP / 不做 mass erase**(超出烧录本固件所需的擦写一律不做)。
8. 改动限本项目目录,别碰 monorepo 其它项目;不启动 MC Workbench/Motor Profiler 等**驱动硬件**的功能(**只读** MCSDK 安装目录里的板卡定义文件是允许的)。
9. **收尾让 MCU 停在安全态**:本诊断固件本身安全(不驱动),但要说明板子下次上电会继续跑它、且它不做任何危险动作。

## §-1 交互协议(HITL)+ 升级机制
**HITL 停等**(须用户手动/授权处):固定输出下块并停,等用户回复才继续:
```
==== 需要你手动操作 [HITL-n] ====
目的:<为什么>
请你做:1) ... 2) ...
做完请回复:"HITL-n 完成"(或反馈实际结果/看到什么)
在你回复前我不继续、不猜测。
====
```
本任务**必有**的 HITL:① 联网下载/装 pip 包授权;② **每次烧录前**确认"授权+母线断+目标板对";③ 让用户**手转转子**并读现象;④ 让用户对照实物/回报观测。**合并同阶段的手动动作,别零碎打断**(仓库"少打断"原则)。

**升级给 CC(区别于 HITL)**:遇到**超出你职责的"需求/硬件详情/安全边界"判断**(如红线#3 的母线供电、引脚映射查不到、疑似要改安全边界),按下块**停止本任务并把球交回 CC**,不要自行决策继续:
```
==== 需 CC 决策 [ESC-n] ====
背景/卡点:...
我已尝试:...(附结果)
涉及的红线/安全问题:...
我需要 CC 定的:...
====
```

- Windows 工具链齐备(F-12):CubeCLT 1.18.0(`STM32_Programmer_CLI` v2.19.0、`arm-none-eabi-gcc` 13.3.1)、CubeIDE 1.19.0(GNU Make)、CubeMX **6.17.0-RC5**、G4 FW V1.6.1/1.6.2、MCSDK 6.4.1。
  - ⚠️ **CubeMX 6.17.0-RC5 headless 生成有已知 bug**(sysmem/syscalls 路径拼接、C_SOURCES 重复/缺续行),`hw_smoke_win/build_win.ps1` 里有绕过办法,**复用它的思路**。

## §3 执行阶段(按序;遇手动/授权/升级走 §-1)
0. §0 权限预检。
1. **先定安全前提**:查 §1b 的霍尔 +5V 来源。**若需母线供电 → ESC 升级 CC,停。** 否则记录"USB 可给霍尔供电",继续。
2. 导出并记录霍尔→G431 引脚映射、霍尔电气类型、VCP UART 及占用情况(附出处)。
3. **设计并生成固件工程**(方案自选,见 §5 自由与留痕):读 3 霍尔,计数 H1(或等效)上升沿 + 记录霍尔状态序列;选定输出通道(见下)。**严守红线#1:不碰任何功率级/ PWM 输出使能。**
4. 编译(复用 Windows 工具链 + RC5 绕过)。产物留磁盘。
5. **HITL:确认授权+母线断+目标板对 → 烧录**本诊断固件(SWD,按 SN)。
6. **HITL:请用户手转转子精确整圈**,读输出;至少测:单霍尔每圈上升沿数(=对极数)、三霍尔一圈总跳变数(=6×对极数),多转几圈复核。
7. 得出极对数结论;若打通了电角度观测,一并演示。
8. 写报告 + 落盘 + commit+push。

**输出通道(自选 + 备选阶梯,选谁/为何写进报告)**:
- 首选 VCP(COM4)串口打印霍尔状态/计数/角度(信息丰富,利于次目标);
- 若 VCP 引脚被占/冲突 → 备选:**LD2(PA5)按 B1(用户键)触发闪 N 下 = 计数**、或 ITM/SWO printf、或调试器实时看全局计数变量。任选其一保证拿到结果。

## §4 失败处理与重试矩阵(关键步骤没达预期时怎么办)
对每个关键步骤:先按下列**依次尝试**;**全部无效**则按该步"升级"栏停下反馈(HITL 或 ESC),**附上每次尝试的现象**,别硬试、别越红线。

| 关键步骤 | 预期 | 若不达预期,依次尝试 | 全部无效 → 反馈 |
|---|---|---|---|
| 霍尔 +5V 来源判定 | 明确 USB 域可供电 | 查 UM2415 供电原理图→查板卡定义→万用表量 J1 +5V 对 GND(经 HITL 请用户量) | **ESC 升级 CC**(涉及母线/安全边界) |
| 霍尔引脚映射 | 3 路确定到 GPIO | UM2415 原理图→MCSDK 板卡定义→生成参考工程比对→**经验探测**(轮询所有候选脚,手转看哪些脚跳变) | ESC 升级 CC,附候选与观测 |
| 霍尔读数 | 手转时电平翻转 | 开内部上拉(开漏)→换确认的引脚→核对霍尔已供电→降速手转→加输入滤波 | HITL 报"读不到",列已试项 |
| 计数不稳/抖动 | 每圈稳定 2 或 4 | 软件去抖/定时器输入滤波→多圈取一致值→三霍尔交叉核对→与齿槽≈24 对照 | HITL 报不稳定 + 原始跳变记录 |
| 极对数结论 | 干净 2 或 4 | 与齿槽(≈22≈24)、三霍尔总跳变、必要时万用表法交叉验证 | 报"结论不唯一"+ 各法数据,请 CC 研判 |
| 输出通道 | 用户能读到结果 | VCP→LD2 闪灯+B1 触发→ITM/SWO→调试器看变量 | HITL 报输出受阻,列已试通道 |
| 烧录 | 写入并运行 | 确认 SN/连接(HOTPLUG/UR)→查线→只读连通性先验证 | HITL 报烧录失败 + 完整命令与报错 |
