## 这是什么项目(一句话)
STM32 **Nucleo-G431RB**(STM32G431RB,Cortex-M4F @170MHz)主控 + **X-NUCLEO-IHM16M1** 官方扩展板上的开发。X-NUCLEO-IHM16M1 是 ST 的**三相电机(BLDC/PMSM)驱动评估板**(IHM 系列),故本项目很可能是**电机控制**方向 —— 但**具体用途与控制方案由用户逐步说明**,落到 [`docs/REQUIREMENTS.md`](docs/REQUIREMENTS.md)。当前处于**需求讨论阶段,尚未实现任何功能**。

## 交互方式:语音听写输入(重要)
本项目的对话**主要由语音听写产生**:用户说话 → 语音转文字(STT)→ 文本发给你,因此你收到的文字**可能带有识别误差**(同音字、词边界错位、标点缺失、专业术语被听错等)。

- **按意图理解,别死抠字面**:遇到读不通或明显不对的词,结合上下文与本项目领域(STM32 嵌入式、电机控制 / FOC / 功率电子 / 传感采样)推断用户真正想说的,心里纠正后再回应。
- **领域聚焦、话题不发散**,绝大多数误识别都能靠上下文确定 —— 大胆按最合理的技术含义解读。
- **少打断**:只有当某处误识别会影响关键决策、且上下文无法确定时,才用一句话简短确认;不要逐字逐句反复追问。

## 工作原则:工程严谨(必守)
- **尊重数学、控制理论与电子 / 电机的事实**:不得为迎合用户而编造不符合科学或事实的回答。不确定就说不确定并给出验证办法,**绝不自信地胡说**;算不出 / 没把握的,讲清依据与边界,不谎报成功。
- **遇失败先查实现,别先怀疑已验证事实**:见 [`docs/FACTS.md`](docs/FACTS.md) 头部规则。

## ⛔ 操作红线(安全 · 必守)
**未经用户在本会话中明确指示,不得:① 开始编码实现、② 烧录(下载固件到 MCU)、③ 给电机扩展板上电机电源 / 驱动电机转动。**
- 原因:开发板常**无人值守**;电机会**物理转动**,堵转、失控、接线错误或参数失当可能损坏硬件、烧驱动级,甚至造成机械危险,用户不在场无法急停。烧录也可能让 MCU 进入异常状态。
- 默认只做:需求 / 设计 / 架构 / 分析 / 文档(本会话主业),无需额外许可。
- **联机有变数**:主机可能接多块 ST-Link / 板子,或一块没连。**烧录或读串口前,先按 [`docs/OPERATIONS.md`](docs/OPERATIONS.md)『设备探测』确认 ST-Link 与目标板在线且确为 G431RB** —— 别假定"连着的就是目标板"、别写死端口 / 探针名。
- 即便烧录,**也先确认电机供电已断 / 电机空载**,代码默认不输出 PWM 占空,再逐步放开。
